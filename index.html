PHISHORA's heuristic analysis checks emails for multiple structural and textual anomalies often found in social engineering campaigns:

  • Urgent Action Signals: Social engineers rely heavily on immediate threat vectors (e.g. suspension, billing crisis). We weight keywords associated with high urgency or immediate action demands.
  • Sender Enclosure Spoofing: Attackers often set their "Display Name" to mirror high-integrity entities (e.g., "Chase Bank Operations") while holding a mismatching domain name enclosed in angle brackets.
  • Numeric Host Signatures: High-integrity web services host their assets under validated domains rather than raw IP addresses inside hyperlinked URLs.
  • Homograph Spoofing (Unicode): Attackers use visually indistinguishable Unicode lookalikes (Cyrillic equivalents) to bypass traditional spam checkers. Mixed-script detection flags URLs with these lookalikes.
  • Formatting Redundancies: Punctuation abuse (e.g., !!!) and excessive use of capitalization mimic aggressive call-to-actions, indicating potential fraud.

Entropy calculations quantify the mathematical randomness and unpredictability of a password string, calculating the search space complexity for an attacker:

  • Shannon Entropy Equation: Expressed as $E = L \times \log_2(R)$, where $L$ defines password character length and $R$ quantifies the size of the symbol pool based on character types present.
  • Symbol Sets Pool Sizes ($R$): Lowercase Letters ($26$), Uppercase Letters ($26$), Numeric Digits ($10$), and Special Symbols ($33$). Including all categories boosts the baseline exponentially.
  • Keyboard Walk Detection: Detects contiguous spatial progressions across a standard QWERTY keyboard layout (e.g. `qwerty`, `asdf`, `1234`). These represent negligible search spaces for custom password dictionary algorithms.
  • Common Dictionary Filtration: High entropy can be spoofed by simple, long sequential phrases. PHISHORA intercepts the top 50 worst-performing passwords to prevent falsified high security classifications.

Secure password compromise calculations leverage advanced Web Cryptography and mathematical range-query interfaces:

  • Local Cryptographic Hashing: Plaintext passphrases entered here are immediately converted to a hexadecimal SHA-1 string inside the browser using standard `SubtleCrypto` digests, never reaching the external network interface in plain format.
  • K-Anonymity Partial Masking: The first 5 characters of the hex hash (representing a pool of $1,048,576$ different possibilities) are transmitted to the API endpoint. HIBP returns a subset of all matching partial suffixes and their leakage frequency.
  • Zero Knowledge Suffix Auditing: The browser parses the returned suffix records locally and checks for a exact suffix match. This prevents any third-party observers from identifying the specific credential checked.
